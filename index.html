Answer templates

Default structure
  1. Restate the problem in one sentence.
  2. Clarify: “I’d rather act with ~80% than wait for 100% and miss timing.”
  3. Apply a strategy (0→100 / QA / Buy-in / Prioritize).
  4. Execute: owners, cadence, artifacts, risk log, milestones.
  5. Optional: a relevant anecdote slice + measurable result.
ARR lever framing

Ending ARR = Beginning ARR + New ARR + Expanded ARR − Churned ARR − Contracted ARR

  • Increase New ARR: top-of-funnel, win rate, ACV
  • Increase Expanded ARR: upsell/cross-sell, reduce contraction
  • Decrease Churned ARR: support, UX, ease-of-use

Core strategies

0 → 100 (Scope → Plan → Execute → Scale)

Scoping / Discovery (5 questions)

  • Where are we?
  • Where do we want to go?
  • Why do we want to go there?
  • Who do we need to consult?
  • What data do we have to inform us?

Planning

  • How do we get there? people, resources, money, time, target audience
  • How do we measure success? financial, efficiency, operational, quality

Execution

  • Kickoff: key contacts, vision, process, next steps, FAQs
  • Prioritize/conflict: tradeoffs → 1-way vs 2-way door checkpoints
  • Accountability/QA: reporting cadence → reflect/correct/reward
  • Scale: documentation, enable people to teach, adjacent impact
Buy-in (Empathize → Equip → Empower)
  • Empathize: understand needs + why they should care
  • Equip: guidelines, trackers, feedback loops
  • Empower: affirm strengths, create safe space for growth
  • Data: make tradeoffs concrete; move with imperfect info

90-day integration

Shorthand flow
  1. Know who reports to whom
  2. Understand current state + gaps vs “us”
  3. Create plan: owners, timeline, budget (with SMEs)
  4. Migrate data, train on tools, sunset duplicates
  5. Measure success → iterate + incentivize

DD red flags

Revenue
  • Revenue concentration
  • High churn
  • Heavy discounting
  • Weak upsell path
  • Renewal terms / downside protections
Security
  • SOC 2 / ISO posture, incident logs
  • Weak access controls
  • Missing encryption / backups
  • Inadequate incident response
People
  • Attrition / morale risk
  • Single points of failure
  • Skill gaps; low training completion
